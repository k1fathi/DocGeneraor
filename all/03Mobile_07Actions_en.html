User Guide for Actions Interface

Main Components

  • ID: Unique identifier for each action.
  • Action Name: Title given to the action.
  • Action Description: Brief explanation of what the action does.
  • Action Selection: Option to choose an action-related category.
  • Event String: The event that triggers the action.
  • Chain Process: Sequence of steps that the action is part of.
  • Notifications: Alerts related to the action.
  • Actions: Edit or delete options for each listed action.

Usage Instructions

Follow these steps to utilize the Actions interface:

  • Log in to the system.
  • Navigate to the Actions section.
  • Use the Word Search bar to locate specific actions.
  • Adjust the Filter Deleted options to view actions.
  • Click the button to create a new action.

Detailed Explanation of Features

  • ID: Automatically generated for every action, ensuring uniqueness.
  • Action Name and Description: Important for user understanding and organization.
  • Action Selection: Allows grouping actions based on their type.
  • Event String: Determines when the action is executed.
  • Chain Process: Useful for tracking the series of actions in workflows.
  • Notifications: Keeps users informed about the status of actions.
  • Actions Column: Icons provide quick access to edit () or delete () actions.
