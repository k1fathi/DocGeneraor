User Guide for Journeys Interface

Main Components

  • ID: Unique identifier for each journey.
  • Name: The title of the journey.
  • Level Name: Indicates the level associated with the journey.
  • Platform: Specifies the platform the journey is designed for (e.g., Web, Mobile).
  • Description: A brief explanation of the journey's purpose.
  • Running Status: Displays the current activity status (e.g., Running, Stopped).
  • Actions: Provides options to interact with each journey (e.g., edit, delete).

Usage Instructions

To Add a New Journey:
  • Click the button at the top right corner of the interface.
  • Fill in the required fields for the journey details.
  • Submit the form to create the journey.

Detailed Explanation of Features

  • Word Search: Helps filter journeys based on keywords in their names or descriptions.
  • Filter Deleted: Allows you to choose whether to show only undeleted journeys or all journeys.
  • Running Status Colors:
    • Running - indicates the journey is currently active.
    • Stopped - indicates the journey is inactive.

Use Case Scenario

Scenario: A user wants to edit the description of an active journey.
  • Navigate to the journey list and locate the desired journey.
  • Click on the Edit icon in the Actions column.
  • Modify the description in the provided text field.
  • Save the changes to update the journey.
